SuperAnim 绑定js过程

Posted on 2015-03-21   |   In cocos2d

google需要的资料,然后参考wiki下载python 依赖库yaml cheetah
命令行:
python -m easy_install pyyaml
python -m easy_install cheetah
搞定.
然后打开cocos2d-x js绑定脚本的test的目录.

  1. 修改user.cfg userconf.ini按照目录配置即可
  2. 运行即可

如果出现 ‘string not found’+
请使用 ndk-android-n9d 来进行编译,修改 test.ini 安卓ndk目录下libstdc++/4.7/目录以及升级了,修改为libstdc++/4.8/ 就可以通过.其他ndk可以继续修改头文件依赖来通过.
只要见到这个

就可以认为通过

另外我对其中的文章中的未详细说明的地方再补充一下.
问题2:里面由于clang升级到了3.3,
修改
clang_flags = -nostdinc -x c++ -std=c++11 为
clang_flags = -nostdinc -x c++ -std=c++11 -U __SSE__
问题3:现在我用的环境不会中断编译,只是生成的api不好导入到ide中,如果不需要补全的同学不需要关系着问题,修改后可以仍然导入ide补全
问题4:
由于嫁接上去的库用到回调函数.
参考帖子类似于callback ,delegate等运行的情况无法做到自动绑定.
因为工具无法做到完全自动绑定，所以必须有一部分手写的（脚本语言都是这样，习惯就好了，谢谢）。其实同学们应该理解为这是一个c++调用js的情况.所以请参考文章,自行对回调编写,也可以参考($工程目录)/frameworks/js-binding/bindings/manual下的写法来编写需要的功能.需要写一些js的接口方便调用.
